《堆叠修罗场》(Mystic Stacker) 项目深度交接文档

1. 项目概览 (Project Overview)

中英文名：堆叠修罗场 / Mystic Stacker

核心定位：一款基于 React 构建的、具有克苏鲁/神秘学叙事色彩的卡牌堆叠自动化模拟游戏。

设计灵感：融合了《堆叠大陆》(Stacklands) 的合成机制与《密教模拟器》(Cultist Simulator) 的叙事氛围。

最终愿景：玩家通过经营微小的劳动力，建立起一套剥削血肉与禁忌知识的工业体系，最终直视真理，达成维度飞升。

2. 需求与系统设计 (System Design)

2.1 核心玩法机制

堆叠匹配 (Stacking)：游戏核心交互。将特定组合的卡牌垂直堆叠，若满足配方条件，将启动计时合成。

底座原则 (Base-Driven)：每堆卡牌的最底层（Index 0）决定了搜索配方的范围。例如：只有以“解剖台”为底座的堆叠才会触发解剖类配方。

复杂优先 (Complexity Priority)：当一堆卡牌满足多个配方时（例如 A+B 和 A+B+C），引擎会优先执行所需的输入卡牌数量最多的配方（A+B+C）。

进度条重置：为了防止玩家利用低级配方白嫖进度时间，一旦堆叠内容发生变化导致触发了不同的配方，进度条将立即强制归零。

2.2 叙事与进阶阶段

觉醒期：建立基础循环（探索者 + 门 -> 梦境 -> 灵视），解决生存劳动力来源。

扩张期：开启外部地图（发现森林/废墟/墓地），开始大规模采集原始物资（木/石/铁/骨）。

工业期：将劳动力转化为“狂信徒”，建造工业化设施（解剖台/炼金釜/印刷机）。

剥削期：开启暗黑流水线。通过解剖凡人获取血液，通过熬制尸体获取灵液，通过压榨缸中脑量产灵视。

飞升期：筑造虚空祭坛，合成终极真理，达成胜利结局。

3. 技术架构与开发规范 (Technical Architecture)

3.1 核心技术栈

React 18：利用 Function Components 和 Hooks 驱动。

Tailwind CSS：用于高性能、响应式的 UI 布局。

Lucide-React：仅限使用库中极其基础的图标（Help, X, Zoom, etc.），防止版本兼容引发的白屏。

原生 Emoji：所有卡牌内容使用 Emoji，确保 0 资源加载失败风险。

3.2 底层性能与稳定性防线（接手必读）

由于本项目运行在严苛的浏览器预览沙盒环境中，曾发生多次崩溃闪退，以下逻辑绝对严禁乱改：

Ref 状态同步锁 (stateRef)：

原理：由于 window 事件监听器（PointerMove/Up）在 React 闭包中容易读取到过时的 State，且高频绑定会导致内存泄漏。

方案：使用 useRef 同步所有关键状态（stacks, zoom, pan）。所有的 DOM 事件处理函数只从 stateRef.current 中读取数据。

O(1) 配方索引映射：

原理：每 50ms 的游戏 Tick 如果遍历 200 个配方会造成严重掉帧甚至白屏。

方案：使用 RECIPE_INDEX 对象。在初始化时按“底座卡牌类型”对配方进行分类，合成时仅查询当前底座对应的 3-5 个潜在配方。

内存防弹：对象计数匹配：

原理：传统的 splice 和 slice 操作在循环中会产生大量垃圾临时数组，触发 GC 抖动导致闪退。

方案：匹配时先统计堆叠中各卡牌类型的数量（availableCounts），直接比对配方的哈希表计数，不产生任何中间数组。

无 UI 列表渲染：

原理：瞬间渲染 200 个带样式的配方节点会导致沙盒渲染超时白屏。

方案：取消配方手册 UI，改为通过进度条上的 recipe.name 和“深渊低语”动态暗示玩家下一步动作。

4. 项目启动与初始化 (Setup)

环境要求：支持 React 运行的沙箱或现代浏览器。

坐标系统：

全局使用“画布坐标系”。

鼠标/触控物理坐标需通过 (clientX - pan.x) / zoom 换算为画布坐标。

启动流：

用户点击开局选单 -> startGame(origin) -> 注入初始卡牌 ID -> 启动 setInterval (50ms)。

6. 后续开发建议 (Maintenance)

卡牌扩展：仅需在 CARD_TYPES 添加定义，注意 bg/border 的 Tailwind 类名需完整。

配方扩展：在 rawData 数组中添加字符串配置，语法为 [名称, 时间, 输入, 保留, 输出]。

UI 优化：如果需要增加左侧侧边栏，务必确保不会在 Game Loop 中触发大规模的 DOM 重排。

Bug 预警：若发现点击卡牌没反应，优先检查 handleCardPointerDown 是否正确穿透了缩放层。
